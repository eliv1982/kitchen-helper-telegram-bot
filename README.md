# Kitchen Helper — Telegram Bot

## Цель проекта

Учебный мини-бот **Kitchen Helper** помогает придумать простые домашние блюда из продуктов, которые есть у пользователя. Проект демонстрирует работу Telegram-бота на **aiogram**, интеграцию с **OpenAI API** и хранение контекста диалога в памяти.

## Стек

- Python 3.10+
- [aiogram](https://docs.aiogram.dev/) 3.x — Telegram Bot API
- [openai](https://github.com/openai/openai-python) — официальный SDK OpenAI
- [python-dotenv](https://github.com/theskumar/python-dotenv) — переменные окружения из `.env`

## Структура файлов

| Файл | Назначение |
|------|------------|
| `bot.py` | Точка входа: команды, обработка сообщений, запуск polling |
| `config.py` | Загрузка настроек из `.env` |
| `context_manager.py` | Контекст диалога в `dict` (один контекст на `user_id`) |
| `api_client.py` | Запросы к OpenAI, system prompt, логирование usage |
| `requirements.txt` | Зависимости проекта |
| `.env.example` | Шаблон переменных окружения (без секретов) |

## Как создать `.env`

1. Скопируйте шаблон:

   ```bash
   copy .env.example .env
   ```

   (в Linux/macOS: `cp .env.example .env`)

2. Заполните обязательные поля:

   - `BOT_TOKEN` — токен от [@BotFather](https://t.me/BotFather)
   - `OPENAI_API_KEY` — ключ с [platform.openai.com](https://platform.openai.com/)

3. При необходимости измените `OPENAI_MODEL`, `TEMPERATURE`, `MAX_TOKENS`.

**Не коммитьте файл `.env` в репозиторий.**

## Установка зависимостей

```bash
python -m venv .venv

# Windows
.venv\Scripts\activate

# Linux / macOS
source .venv/bin/activate

pip install -r requirements.txt
```

## Запуск бота

```bash
python bot.py
```

В логах появится сообщение о старте бота. Откройте своего бота в Telegram и отправьте `/start`.

## Как проверить контекст

1. Напишите боту: «У меня есть курица, рис, яйца и сыр. Что можно приготовить?»
2. Уточните: «А если без духовки?» — бот должен помнить перечисленные продукты.
3. Уточните: «Хочу самый быстрый вариант.» — бот должен учитывать и продукты, и ограничение без духовки.
4. В консоли смотрите логи: `context_len` растёт с каждым обменом сообщениями.

Бот должен продолжать опираться на ранее указанные продукты и уточнения в рамках одного диалога.

Контекст хранится только в оперативной памяти процесса и сбрасывается при перезапуске бота.

## Как очистить контекст

- Команда `/reset`
- Сообщение: `очистить контекст` (регистр не важен)

После очистки бот не помнит предыдущие реплики в рамках этого `user_id`.

## Таблица экспериментов

Для каждого прогона менялась только `TEMPERATURE` в `.env` (после перезапуска бота). Сценарий: пользователь указывает продукты (курица, рис, яйца, сыр), уточняет «А если без духовки?», затем «Хочу самый быстрый вариант.»; после `/reset` — «А теперь предложи что-нибудь из тех продуктов.»

Токены указаны суммарно по всем OpenAI API-запросам внутри одного сценария, кроме команды /reset.

| model | temperature | max_tokens | run | effect | input_tokens | output_tokens | total_tokens | approximate_cost |
|-------|-------------|------------|-----|--------|--------------|---------------|--------------|------------------|
| gpt-4.1-mini | 0.2 | 600 | 1 | Ответы более строгие и предсказуемые. Бот даёт простые варианты, меньше фантазирует, формулировки компактные и практичные. | 741 | 232 | 973 | < $0.01 |
| gpt-4.1-mini | 0.7 | 600 | 2 | Самый сбалансированный вариант: ответы короткие, естественные, достаточно практичные, контекст учитывается без лишних деталей. | 702 | 192 | 894 | < $0.01 |
| gpt-4.1-mini | 1.0 | 600 | 3 | Ответы более развёрнутые и вариативные. Бот предлагает больше деталей, но ответы становятся менее компактными. | 901 | 345 | 1246 | < $0.01 |

*approximate_cost* — по тарифам OpenAI на момент эксперимента (см. [pricing](https://openai.com/api/pricing/)).

## Краткий вывод по экспериментам

- Бот успешно подключается к OpenAI API и отвечает на сообщения пользователя.
- Контекст работает: после уточнений «А если без духовки?» и «Хочу самый быстрый вариант.» бот продолжает учитывать ранее названные продукты.
- Команда `/reset` корректно очищает контекст. После очистки бот больше не помнит старые продукты и просит перечислить их заново.
- При temperature=0.2 ответы более строгие и предсказуемые.
- При temperature=0.7 ответы наиболее сбалансированные для бытового помощника.
- При temperature=1.0 ответы более вариативные и развёрнутые, но менее компактные.
- Для этого учебного сценария оптимальным выглядит режим temperature=0.7 и max_tokens=600.
