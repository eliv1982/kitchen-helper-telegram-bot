# Сравнение prompt v1 и prompt v2

Сравнение по фактическим файлам [response_v1.json](./response_v1.json) и [response_v2.json](./response_v2.json) после запуска `run_prompts.py` (gpt-4o-mini, temperature=0.2).

## Соответствие JSON-формату

| Критерий | v1 | v2 |
|----------|----|----|
| Валидный JSON | Да | Да |
| Схема `title` / `steps` / `notes` | Да | Да |
| Без Markdown вне JSON | Да | Да |

Оба ответа прошли `json.loads` в скрипте. Формат соблюдён в обоих случаях; разница — в содержании и управляемости.

## Полезность steps

**v1** («Запеченная курица с картофелем и морковью»): 7 шагов — нарезка, форма, духовка 40–50 мин и корочка без фольги. Курица, картофель, морковь, лук и сметана есть в шагах (сметана в шаге перемешивания перед запеканием). Минус: выбрана **духовка** и фольга — для «простого ужина без сложных техник» это менее предсказуемо, потому что v1 не задаёт технику и ограничения.

**v2** («Курица с картофелем и морковью в сметанном соусе»): 7 шагов на сковороде — нарезка, обжарка курицы, лук, овощи, **сметана с водой при тушении 20 минут**. Название совпадает с шагами. Все пять продуктов явно в `steps`; лук и сметана в процессе готовки, не в `notes`.

**Вывод:** v2 даёт более предсказуемый и воспроизводимый план под сценарий Kitchen Helper; v1 формально подробнее, но менее управляем по технике и доп. советам.

## Лаконичность notes (до 3 тезисов)

**v1:** 2 пункта в `notes`, оба уводят от задания: «любимые специи для аромата», «зелень или свежие овощи» при подаче — это доп. ингредиенты, не закреплённые в промпте v1.

**v2:** ровно 3 коротких пункта — огонь («чтобы не подгорело»), проверка готовности картофеля и курицы, долив воды. Без специй/зелени и без переноса обязательных продуктов в `notes`.

**Вывод:** v2 соблюдает лимит и назначение `notes`; v1 использует `notes` для доп. ингредиентов и украшения подачи.

## Итоговый вывод

| | v1 | v2 |
|---|----|----|
| JSON | Валиден | Валиден |
| Управляемость | Ниже: духовка, специи и зелень в notes | Выше: сковорода/тушение, все продукты в steps, title ≈ рецепт |
| Для Kitchen Helper | Слабее как шаблон бота | Лучше подходит как основа функции |

Промпт **v2** с явными ограничениями даёт ответ, который проще парсить и показывать пользователю в Telegram-боте. **v1** полезен как контраст: при минимальном промпте модель всё ещё возвращает JSON, но содержание менее предсказуемо.
