# Домашнее задание: промпт-инжиниринг и JSON (Kitchen Helper)

## Цель задания

Отработать составление и улучшение промпта для LLM с жёстким форматом ответа (валидный JSON), сравнить качество ответов v1 и v2 и подготовить артефакт, пригодный как основа функции бота Kitchen Helper.

## Бытовая мини-задача

**Kitchen Helper** — составить простой план ужина из продуктов, которые уже есть дома:

курица, картофель, морковь, лук, сметана.

## Параметры запуска

| Параметр | Значение |
|----------|----------|
| Инструмент | Python script from Cursor terminal |
| Модель | gpt-4o-mini |
| Temperature | 0.2 |
| Response format | json_object |
| Max tokens | 600 |

Запуск:

```bash
python homework/prompt_json_kitchen_helper/run_prompts.py
```

Артефакты прогона:

- [run_log.md](./run_log.md) — параметры, файлы, статус валидации JSON
- [run_transcript.md](./run_transcript.md) — полный транскрипт «промпт → ответ» для отчёта и скриншота

## Промпт v1

Файл: [prompt_v1.txt](./prompt_v1.txt)

Базовая схема: роль → контекст → задача → формат ответа. Минимум ограничений на содержание, технику и использование ингредиентов.

## Итерация качества

По ответу на v1 ([response_v1.json](./response_v1.json), получен при запуске `run_prompts.py`) выявлены проблемы:

- модель выбрала запекание в духовке (и фольгу), хотя в v1 просили «простой ужин без сложных техник», но техника не закреплена;
- в `notes` — «любимые специи» и «зелень или свежие овощи», то есть советы про доп. ингредиенты/подачу, а не только техника;
- базовый промпт не закрепил лимит `notes`, запрет докупать продукты и явные требования к использованию всех ингредиентов в `steps`.

Улучшения внесены в **prompt v2**: уточнённая роль, ограничения по ингредиентам и технике, few-shot, лимит `notes`, требование использовать все пять продуктов в `steps` (включая лук и сметану в процессе готовки).

## Итоговый JSON-артефакт

Файлы [response_v1.json](./response_v1.json) и [response_v2.json](./response_v2.json) **получены фактическим запуском** `run_prompts.py`, без ручного редактирования.

Итоговый ответ v2 (заголовок и структура):

```json
{
  "title": "Курица с картофелем и морковью в сметанном соусе",
  "steps": ["...", "..."],
  "notes": ["...", "..."]
}
```

Сравнение версий: [comparison.md](./comparison.md).

## Краткий вывод

Пара **prompt_v2 + response_v2** даёт более управляемый и практичный план ужина в фиксированной JSON-схеме: сковорода/тушение, все заданные продукты в шагах, короткие `notes`. Это можно использовать как основу будущей функции Telegram-бота Kitchen Helper: пользователь перечисляет продукты → бот отправляет улучшенный prompt → ответ парсится как JSON → в Telegram показываются заголовок, шаги и заметки.
