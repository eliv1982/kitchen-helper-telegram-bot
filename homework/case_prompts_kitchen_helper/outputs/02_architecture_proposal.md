# Architecture Proposal: Kitchen Helper

## Дерево файлов

```plaintext
kitchen_helper/
│
├── bot.py                # Основной файл для запуска бота
├── config.py             # Конфигурация и настройки
├── handlers/             # Обработчики команд и сообщений
│   ├── __init__.py
│   ├── commands.py       # Обработчики команд (например, /start, /reset)
│   └── messages.py       # Обработчики текстовых сообщений
│
├── services/             # Сервисы для работы с API и логикой
│   ├── __init__.py
│   ├── recipe_service.py  # Логика получения рецептов из OpenAI API
│   └── context_manager.py  # Управление контекстом диалога пользователя
│
├── models/               # Модели данных
│   ├── __init__.py
│   └── user_context.py    # Хранение контекста пользователя
│
├── tests/                # Тесты
│   ├── __init__.py
│   ├── test_handlers.py   # Тесты для обработчиков
│   └── test_services.py    # Тесты для сервисов
│
├── requirements.txt      # Зависимости проекта
└── README.md             # Документация проекта
```

## Описание модулей

### `bot.py`
Основной файл, который инициализирует бота, загружает конфигурацию и запускает обработчики.

### `config.py`
Содержит настройки, такие как токены для Telegram и OpenAI API, а также другие конфигурационные параметры. Использует библиотеку `dotenv` для загрузки переменных окружения.

### `handlers/`
Содержит обработчики команд и сообщений, которые отвечают за взаимодействие с пользователем.

- **`commands.py`**: Обработчики команд, таких как `/start` и `/reset`, которые инициализируют диалог и сбрасывают контекст.
- **`messages.py`**: Обработчики текстовых сообщений, которые обрабатывают запросы пользователя о рецептах и идеях.

### `services/`
Сервисы, которые содержат бизнес-логику и взаимодействуют с внешними API.

- **`recipe_service.py`**: Логика для запроса рецептов из OpenAI API на основе введенных пользователем продуктов и предпочтений.
- **`context_manager.py`**: Управляет контекстом диалога пользователя, хранит и восстанавливает состояние между сообщениями.

### `models/`
Содержит модели данных, которые используются в проекте.

- **`user_context.py`**: Хранит информацию о контексте пользователя, включая последние запросы и предпочтения.

### `tests/`
Содержит тесты для различных компонентов проекта.

- **`test_handlers.py`**: Тесты для проверки корректности работы обработчиков команд и сообщений.
- **`test_services.py`**: Тесты для проверки логики сервисов, таких как получение рецептов и управление контекстом.

### `requirements.txt`
Список зависимостей проекта, включая `aiogram`, `openai`, `python-dotenv` и другие необходимые библиотеки.

### `README.md`
Документация проекта, включая инструкции по установке, запуску и использованию бота.

## Хранение контекста
Контекст пользователя будет храниться в памяти во время работы бота. Для этого используется класс `UserContext` в `models/user_context.py`, который будет обновляться при каждом взаимодействии с пользователем. В будущем можно рассмотреть возможность сохранения контекста в базе данных или файле для более устойчивого хранения.

## Будущие избранные рецепты
В будущем можно добавить функциональность для сохранения избранных рецептов. Это может быть реализовано через дополнительный метод в `context_manager.py`, который будет хранить идентификаторы избранных рецептов в контексте пользователя.

## Тесты
Тесты будут написаны с использованием библиотеки `unittest` или `pytest`. Они будут проверять корректность работы обработчиков и сервисов, а также взаимодействие с OpenAI API.

## Конфигурация
Конфигурация будет храниться в `config.py`, где будут загружены токены и другие параметры из файла `.env`. Это обеспечит безопасность и удобство в управлении настройками проекта.

---

Эта структура кода обеспечивает простоту, понятность и возможность дальнейшего роста проекта, позволяя сосредоточиться на разработке функционала без избыточной сложности.
