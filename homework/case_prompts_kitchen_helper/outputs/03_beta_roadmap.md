# Beta Roadmap: Kitchen Helper

## Этапы разработки

### Этап 1: Определение функционала
- **Задачи:**
  - Сформулировать основные команды бота (например, /start, /reset, запрос рецепта).
  - Определить формат ввода от пользователя (продукты, тип блюда).
  - Установить ограничения на типы рецептов (без медицинских рекомендаций).

### Этап 2: Реализация базового функционала
- **Задачи:**
  - Реализовать обработку команд бота.
  - Создать механизм хранения контекста диалога.
  - Реализовать генерацию простого рецепта или кулинарной идеи на основе запроса пользователя.
  - Реализовать сброс контекста через команду /reset.

### Этап 3: Улучшение UX
- **Задачи:**
  - Создать простую и понятную структуру ответов бота.
  - Добавить возможность уточнения запроса (завтрак, обед, ужин и т.д.).
  - Реализовать возможность сохранения понравившихся рецептов (временное хранение в памяти бота).

### Этап 4: Тестирование и отладка
- **Задачи:**
  - Провести тестирование функционала с одним реальным пользователем.
  - Собрать обратную связь и внести необходимые правки.
  - Подготовить документацию по использованию бота.

## Приоритеты задач

- **Must:**
  - Реализация базового функционала: обработка команд и генерация простого рецепта или кулинарной идеи.
  - Сохранение контекста диалога и сброс через /reset.
  
- **Should:**
  - Улучшение UX (структура ответов, уточнение запроса).
  - Временное сохранение понравившихся рецептов.

- **Nice:**
  - Улучшение качества подсказок для LLM-помощника по бытовым кулинарным запросам.
  - Добавление дополнительных команд и возможностей.

## Риски
- Ограниченное время на разработку может привести к недоработке функционала.
- Непредвиденные сложности с обработкой запросов пользователя.
- Возможные ошибки в генерации простого рецепта или кулинарной идеи на основе запроса пользователя.

## Критерии готовности
- Бот корректно обрабатывает команды и запросы пользователя.
- Пользователь может получить рецепт на основе введенных продуктов.
- Контекст диалога сохраняется и сбрасывается по команде.
- Бот предоставляет понятные и простые ответы.

## Минимальный план тестирования
- Тестирование основных команд бота.
- Проверка корректности ответов на различные запросы.
- Оценка удобства взаимодействия с ботом.

## Первые задачи для старта работы
1. Определить команды бота и формат ввода.
2. Реализовать обработку команды /start и /reset.
3. Реализовать генерацию простого рецепта или кулинарной идеи на основе введенных данных.
4. Создать структуру ответов для бота.
